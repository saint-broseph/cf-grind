# 1857C_Assembly_via_Minimums.md

**Problem Link:** [https://codeforces.com/problemset/problem/1857/C](https://codeforces.com/problemset/problem/1857/C)

### 1. Problem Summary
An array $a$ of $n$ integers was used to construct a new array $b$ of size $\frac{n(n-1)}{2}$, containing $\min(a_i, a_j)$ for all pairs $(i, j)$ where $i < j$. Given array $b$, reconstruct any possible original array $a$.

### 2. Core CS/DSA Concepts
Combinatorics, Sorting, Greedy Construction.

### 3. Core Mathematical/Algorithmic Logic
Consider the original array $a$ in sorted order: $a_1 \le a_2 \le \dots \le a_n$.
* The smallest element $a_1$ will be the "minimum" in pairs with $a_2, a_3, \dots, a_n$. This results in **$(n-1)$** occurrences of $a_1$ in array $b$.
* The second smallest element $a_2$ will be the "minimum" in pairs with $a_3, a_4, \dots, a_n$. This results in **$(n-2)$** occurrences of $a_2$ in array $b$.
* This pattern continues: the $i$-th element in sorted order contributes to exactly **$(n-i)$** elements in array $b$.
* The largest element $a_n$ never appears as the "minimum" of a pair (unless it's equal to $a_{n-1}$), so we can just pick a very large value (like $10^9$) for $a_n$.



### 4. Time & Space Complexity
* **Optimal:** $O(M \log M)$ where $M = \frac{n(n-1)}{2}$. The bottleneck is sorting array $b$. Since $n \le 1000$, $M$ is roughly $5 \cdot 10^5$, which fits well within 2 seconds.

### 5. Pseudocode
```text
For each test case:
    Read n
    M = n*(n-1)/2
    Read array b of size M
    Sort(b)
    
    Current_Index = 0
    Result_Array = []
    
    For i from 1 to n-1:
        // The i-th smallest element appears (n-i) times in sorted b
        Append b[Current_Index] to Result_Array
        Current_Index += (n - i)
        
    // Append the n-th element (anything >= last element)
    Append 10^9 to Result_Array
    Print Result_Array
```

### 6. Actual C++ Code
``` cpp
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int n;
    cin >> n;
    int m = n * (n - 1) / 2;
    vector<int> b(m);
    for (int i = 0; i < m; i++) cin >> b[i];

    // Sorting allows us to pick the smallest available mins in blocks
    sort(b.begin(), b.end());

    int current = 0;
    // We only need to find n-1 elements from b
    for (int i = 1; i < n; i++) {
        cout << b[current] << " ";
        // Skip (n-i) occurrences of the current minimum
        current += (n - i);
    }
    
    // The n-th element can be the same as the (n-1)-th or the max possible value
    cout << 1000000000 << "\n";
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    int t;
    cin >> t;
    while (t--) solve();
    return 0;
}
```

### 7. The Key Pattern Learned
**Combinatorial Frequency Mapping:** In problems where an output is generated by all-pairs operations, sort the input and analyze the frequency of contribution. If $a_i$ is the $i$-th element in a sorted sequence, its contribution to a "min" operation is always tied to its rank.

### 8. Key Math Learned
**Handshake Lemma Inverse:** Array $b$ has size $\binom{n}{2}$. The number of pairs involving a specific element as the minimum decreases linearly ($n-1, n-2, \dots, 0$) as we move through the sorted original array.

### 9. Identification in Future Problems
* **Keywords:** "all pairs $(i, j)$", "minimum of pairs", "reconstruct original array".
* **Structural Triggers:** A larger array generated from a smaller one via pairwise operations (min, max, sum).
* **Constraint Triggers:** $n=1000$ and $b$ size $\approx 5 \cdot 10^5$ suggests sorting the large array is fine.
